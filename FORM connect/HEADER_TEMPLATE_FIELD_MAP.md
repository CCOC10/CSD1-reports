# Header Template Field Map

Base template:
- `Header template.pdf`
- Path: `/Users/mrchp/Library/Mobile Documents/com~apple~CloudDocs/WEB dev/CSD1 report/FORM connect/Header template.pdf`

## Coordinate System

- Page size: `A4 portrait`
- Working size: `595 x 842 pt`
- Origin: `top-left`
- If using a PDF library with `bottom-left` origin, convert with:
  - `pdf_y = 842 - top_y - box_height`

Blue text in the design mock means `dynamic field only`.
Final output should render:
- `black` for normal text
- `red` for urgent/warning text only

## Zone Spec

### 1. Fixed Template Zone

This is already baked into `Header template.pdf`.

```json
{
  "fixed_template_zone": {
    "x": 0,
    "y": 0,
    "w": 595,
    "h": 318
  }
}
```

Contains:
- Garuda
- warning box
- urgent marker
- title
- agency name
- issue place line
- case frame
- legal basis line

### 2. Generated Flow Zone

Everything below the legal basis line is generated by the renderer.

```json
{
  "generated_flow_zone": {
    "x": 54,
    "y": 276,
    "w": 487,
    "h": 550
  }
}
```

Order inside generated flow:
1. `หมายถึง` block
2. `ด้วยเหตุ` block
3. intro body
4. account list / request list
5. closing block
6. signature block
7. contact block

Renderer rule:
- do not pin the closing block to a fixed Y
- render it after the dynamic list using `currentY`
- reject output if total generated height exceeds `generated_flow_zone`

## Shared Header Field Map

These fields are shared across the system because the header layout is common.

| field | x | y | w | h | align | font |
|---|---:|---:|---:|---:|---|---|
| `doc_number_line` | 92 | 82 | 150 | 24 | left | TH Sarabun 16pt |
| `issue_date_full` | 174 | 156 | 252 | 20 | center | TH Sarabun 16pt |

Notes:
- `doc_number_line` should render as full line, for example `ที่ ตช 0026.21/2342`
- `issue_date_full` should render as full Thai date, for example `13 เดือน มีนาคม พุทธศักราช 2569`
- recipient, address, case headline, and all body text are no longer shared header fields in the new form
- `หมายถึง` and `ด้วยเหตุ` are generated blocks, not template placeholders

Rules:
- body text defaults to `left aligned`
- do not force distributed/justify layout
- if a line cannot be distributed cleanly, keep it left aligned

## Generated Flow Layout

### Intro Body

Start point:

```json
{
  "body_start": {
    "x": 54,
    "y": 276,
    "w": 487
  }
}
```

Recommended first generated blocks:
- `labelLines` for `หมายถึง`
- `labelParagraph` for `ด้วยเหตุ`
- then render normal paragraph/list blocks below

Suggested label block config:

```json
{
  "label_block": {
    "label_x": 60,
    "label_width": 34,
    "body_x": 108,
    "body_width": 432,
    "gap_after": 4
  }
}
```

### Dynamic Account / Request List

Render this as a vertical list using `currentY`.

```json
{
  "account_list": {
    "start_x": 74,
    "start_y": 392,
    "content_w": 450,
    "number_col_w": 18,
    "line_height": 16.5,
    "item_gap": 6,
    "account_subline_indent": 34
  }
}
```

Recommended rendering pattern for bank requests:
- item 1 = account opening / account identity
- under item 1 render one or more account lines:
  - `เลขที่บัญชี ... ชื่อบัญชี ...`
- item 2+ can reference `ตามข้อ 1`

Example account subline:

```text
เลขที่บัญชี 727278272 ชื่อบัญชี 782727272727
```

### Floating Closing Block

This block has fixed wording but variable Y.

```json
{
  "closing_block": {
    "gap_before": 10,
    "x": 110,
    "w": 430,
    "estimated_height": 62
  }
}
```

Rule:
- render after the dynamic list
- `closing_y = currentY + gap_before`

### Signature Block

```json
{
  "signature_block": {
    "gap_before": 14,
    "x": 315,
    "w": 175,
    "estimated_height": 86
  }
}
```

Recommended contents:
- rank line
- signature image
- signer name
- signer position

### Contact Block

```json
{
  "contact_block": {
    "gap_before": 18,
    "x": 54,
    "w": 487,
    "estimated_height": 34
  }
}
```

Example:
- `{{leaderName}} สวัตร กองกำกับการ 1 กองบังคับการปราบปราม`
- `โทร : {{leaderphoneno}} E-mail : csd1-10@cib.go.th`

## Preflight Rule

The generated area must fit on one page.

```json
{
  "flow_bottom_limit": 806
}
```

Validation formula:

```text
body_height
+ closing_block_height
+ signature_block_height
+ contact_block_height
+ all_gaps
<= 806 - 346
```

If it exceeds:
- do not shrink the font automatically
- do not push content off-page
- reject with `ข้อมูลเกิน 1 หน้า กรุณาแยกหมาย`

## Practical Limits

For `bank-general`, keep these limits in the first implementation:

```json
{
  "max_accounts": 3,
  "max_recipient_address_lines": 2,
  "max_case_headline_lines": 1,
  "max_account_subline_lines_per_account": 1
}
```

If the content is longer:
- split into another request
- or shorten normalized text before render

## Renderer Recommendation

Recommended shared structure:

```text
base template
-> render shared header fields
-> render body intro
-> render dynamic list
-> render floating closing block
-> render signature block
-> render contact block
-> preflight check before save
```
